# Спецификация: Клик по имени в чате + Описание смены

## Фича 1: Клик по имени в чате → профиль

**Цель:** Позволить пользователям переходить на профиль собеседника из чата.

**Поведение:**
- Имя отправителя в сообщении чата (templates/chat.html) становится ссылкой.
- Ссылка ведёт на `/profile/@username`.
- Открывается в том же окне (обычный переход, SPA-навигация отрабатывает).
- Доступно всем ролям.

**Файлы:**
- `templates/chat.html` — обернуть отображаемое имя отправителя в `<a>`.
- Убедиться, что `sender_username` доступен в контексте сообщения.

## Фича 2: Описание конкретной смены (слота)

**Цель:** Добавить текстовое описание к каждому слоту (направление + дата + время).

**Поведение:**
- Новое поле `description` (Text, nullable) в модели `Slot`.
- Отображается на странице `/schedule/{direction_id}/{date}` (slots.html) между прогресс-баром и списком волонтёров.
- Редактирование:
  - **Админ / Лотос** — любой слот.
  - **Руководитель** — только слоты своих направлений (проверка через `DirectionLeader`).
  - **Волонтёр** — только просмотр.
- Inline-редактирование: textarea + "Сохранить" / "Отмена".
- API: `POST /api/slot/{slot_id}/description`.

**Файлы:**
- `models.py` — добавить `description = Column(Text, nullable=True)` в `Slot`.
- Alembic миграция.
- `routers/schedule.py` — добавить `update_slot_description` endpoint.
- `templates/slots.html` — показать описание + форму редактирования при наличии прав.
- `tests/` — добавить тесты.

## Права на профиль (проверено)

- Волонтёры не могут редактировать чужие профили (`can_edit` = False для ROLE_VOLUNTEER).
- Кнопка "Редактировать профиль" скрыта для волонтёров на чужих профилях.
- Админ-роут `/admin/user/{id}/edit` защищён `require_role(..., ROLE_ADMIN)`.
