# Спецификация: Браузерные уведомления и напоминания

## Ограничения

- Приложение работает в offline-лагере: нет интернета, есть только локальная сеть с сервером.
- Настоящие Web Push (VAPID) невозможны без интернета.
- Отложенные уведомления (Notification Triggers API, Alarm API) в чистом вебе не поддерживаются.
- Основной механизм — **in-page уведомления**, когда приложение открыто.

## Цель

Показывать пользователю напоминания о предстоящих сменах и новых сообщениях чата, пока приложение открыто. Дополнительно — Service Worker fallback при восстановлении связи с локальным сервером.

## Функциональность

### 1. Напоминания о сменах

- JS каждую минуту проверяет предстоящие смены текущего пользователя.
- За **24 часа** и за **2 часа** до смены показывается:
  - Toast внутри приложения (`showToast`)
  - Системное уведомление через `new Notification(...)`, если есть разрешение
- Дублирование предотвращается через `localStorage` (храним ID показанных напоминаний).

### 2. Уведомления чата

- Если пользователь не в чате и приходит новое сообщение:
  - Показываем toast «Новое сообщение от Имя».
  - Обновляем badge на иконке «Чат» в нижней навигации.
- Badge обновляется через существующий SSE `/sse/notify`.

### 3. Service Worker fallback

- SW обрабатывает события `push` и `sync`.
- При восстановлении связи с локальным сервером SW может показать накопившиеся уведомления.
- Это не гарантирует точное время, но помогает при закрытой вкладке и восстановлении сети.

### 4. Управление разрешениями

- В профиле пользователя кнопка «🔔 Включить уведомления» / «🔕 Уведомления включены».
- При включении запрашиваем `Notification.requestPermission()`.
- Если отказал — больше не спрашиваем.

## API

- `GET /api/my-upcoming-shifts` — ближайшие смены текущего пользователя
  - Ответ: `[{"slot_id": 1, "direction": "Кухня", "date": "2026-07-09", "time": "10:00", "datetime_iso": "..."}, ...]`
- `GET /api/unread-chat-count` — количество непрочитанных сообщений
  - Ответ: `{"unread": 5}`

## UI

- Toast-уведомления в существующем стиле (`showToast` в `base.html`).
- Кнопка в профиле для включения/выключения уведомлений.
- Badge на иконке чата в нижней навигации (уже есть, расширяем логику).

## Файлы

- `templates/profile.html` — кнопка управления уведомлениями
- `templates/base.html` — глобальный JS для напоминаний и чата
- `static/sw.js` — обработка push/sync событий
- `routers/profile.py` — `/api/my-upcoming-shifts` и `/api/unread-chat-count`
- `tests/test_notifications.py` — тесты API

## Тестирование

- API возвращает корректные upcoming shifts.
- API возвращает корректный unread count.
- Проверка прав доступа (только авторизованные пользователи).
